Dependencies and Installation

This tracking task should run on Python 2.7.9 and above. Once you have Python, you can download the repository with git via: 
$ git clone https://github.com/karasinski/tracking.git
$ cd tracking
The best way to install the project's dependencies is using pip. Once you have pip you can run: 
$ pip install matplotlib numpy pygame seaborn
If you're interesting in running the data analysis scripts you'll also need to install pandas by: 
$ pip install pandas
Once this is finished you can run the experiment by: 
$ python tracking.py
The Subject can then control the start of each trial by pressing the spacebar.

Trial Parameters

Running tracking.py will read the trials indicated in the trials.py file. Each trial is defined by a dict, which has several optional keys. These keys (and their default values) are:
key value
use_joystick True
history 0.
preview 0.
span 1
funckwds {}
length 30
FPS 60
feedback False
invert True
secondary_task False
timer False
The joystick parameter enables the joystick for control of the pointer. Setting this to False allows you to use the mouse instead. Note that this program has been configured to work with a 3Dconnexion SpaceMouse, and that the axis value in the Joystick class may need to be modified to work with other joysticks.

The history parameter sets the amount of the Subject's path that is shown. This value can range from 0 to 1, where 0 shows no history and 1 shows the full window's width of history. The preview parameter is similar to the history parameter, but instead modifies the amount of guidance to be shown to the Subject. Again, a 0 would show no upcoming guidance, while a 1 would show the full window's width of guidance.

The span parameter adjusts the number of seconds of data in the rolling window stoplight metric. A value around 1 second seems to give a reasonable response. Less than this value makes the feedback too noisy, while a larger value creates too much lag.

The funckwds parameter allows you to pass optional arguments into the guidance creation. It's probably best if this option isn't touched. The length parameter adjusts the number of seconds that the trial lasts for (setting this value too high will break the program, but values of >240 will bore the subjects). The FPS parameter adjust the framerate, but this value should also not be touched. matplotlib's FuncAnimation function is at the heart of setting the framerate, so it's doubtful I can do much. Still, feel free to contact me if you're seeing obvious framerate issues.

The feedback parameter turns on the stoplight feedback is set to True. The invert parameter will invert the joystick if modified.

The secondary_task and timer parameters enable secondary tasks if their values are set to True. The secondary_task parameter enables the colored light response used in the lunar lander experiment. Pressing left on the keyboard will turn off a blue light, while pressing right on the keyboard will turn off a green light. These task will be further documented at a later date.
